这份文档只讲我们自己要走的路。四条原则贯穿全文，且都已经写进代码、可被任何人审计：
  1. 开源可审计——所有关键参数、分配比例、罚没规则都以链上常量或创世校验的形式固化，任何人都能在源码里读到、在链上验证到。
  2. 参数写代码——白皮书里出现的每一个数字都对应一行真实的 Go 常量，本文不写"愿景数字"，只写"代码数字"。
  3. 链上求真——没有实现的功能标注为"规划中"，绝不把设想写成既成事实。
  4. 共识共生——MC 的激励来自真实的设备贡献与网络安全，设有社区推荐激励体系，推荐奖励从设备激励池额外拨付，不从被推荐人收益中扣除，保持'激励来自真实贡献'的根本原则。
