# MC（MobileChain）代币分配设计分析

> 版本：v1（2026-07-16 研讨定稿）
> 定位：本文档用于记录 MC 公链代币分配的**设计思路、行业参考、终版比例与落地机制**，供团队回归查阅与治理讨论。
> 原则锚定（MC 自身规矩，不盲从任何一条链）：**开源可审计、参数写代码、链上求真、共识共生**。行业数据仅作设计参考。

---

## 0. 一句话结论

MC 是**固定总量 10 亿 MC、创世一次性铸满、无通胀**的手机全节点公链。代币分配必须把"设备激励"放在最大份额以驱动手机节点参与，同时必须专设"质押安全池"弥补固定总量下无通胀增发的结构性缺口。终版推荐 **5 池：设备激励 55% / 质押安全 15% / 团队 12% / 基金会 13% / 早期开发 5%**。

---

## 1. 行业参考：ETH / DOT / SOL 的公开分配

> 三家均为**通胀模型**，靠持续增发给验证人发奖励——这与 MC 的固定总量模型根本不同，分配逻辑不能直接照搬，只能借鉴其"社区导向 + 团队克制 + 多年释放"的思路。

| 链 | 总量模型 | 主要分配（公开口径） |
|---|---|---|
| **以太坊 ETH** | 创世 7200 万预挖，无硬顶（后期通胀 + EIP-1559 部分通缩） | 2014 公募 ~83.5% / 早期贡献者 ~9.9% / 以太坊基金会 ~9.9% |
| **波卡 DOT** | 初始 10M → 重计 1B，通胀无硬顶 | 历次销售 ~50% / Web3 基金会 ~30% / 预留（团队+未来销售）~20% |
| **Solana SOL** | 初始 500M，通胀无硬顶 | 社区储备 38.89% / 团队 12.79% / 基金会 10.46% / 种子 16.23% / 创世轮 12.92% / 验证人轮 5.18% / 战略 1.88% / 公开拍卖 1.64% |

**从三家提炼的 5 条可吸收经验：**

1. **社区/网络导向份额占大头**：SOL 社区储备 38.89%、DOT 约半数售投资者。MC 的"社区"就是手机节点，设备激励应做全行业罕见的高占比。
2. **团队要克制才有信号**：三家团队+基金会合计约 16%~23%。MC 团队 12% 优于 SOL。
3. **必须有多年线性释放 / cliff**：不锁仓 = 砸盘。MC 每一块都需释放曲线。
4. **集中金库必须上链治理**：DOT 有链上金库、SOL 有基金会治理。MC 基金会须走治理，否则违反"开源可审计"原则。
5. **通胀模型的教训**：三家靠增发养 staking；MC 无增发 → 质押安全必须预先留池。

## 2. MC 的固定约束（分配不能忽略）

| # | 约束 | 对分配的含义 |
|---|---|---|
| 1 | 总量锁死 **10 亿 MC**，创世一次性铸满，`x/mint` 通胀 = 0 | 没有"持续增发"这条收入，一切激励来自预设池 |
| 2 | 验证人/手机节点质押奖励 = 手续费 + 预先留的质押池 | 质押池是唯一的安全资金来源，必须留够 |
| 3 | 早期链上交易少 → 手续费 ≈ 0 | 启动期质押安全 100% 靠质押池 |
| 4 | 设备激励若一次性放完 = 瞬间巨抛压 + 大设备方通吃（中心化） | 设备激励必须按真实任务线性释放 + 锁仓 |

## 3. MC 当前链上实际分配（代码现状）

读 `x/tokenomics/types/keys.go` + `genesis.go`，当前为创世一次性铸满 1e9 MC，分 **3 池**：

| 池 | 占比 | 金额 | 释放/控制 |
|---|---|---|---|
| 团队 Team | 15% | 1.5 亿 MC | 3-of-5 多签，1yr cliff + 3yr 线性 |
| 社区 Community | 35% | 3.5 亿 MC | 治理模块账户（链上治理支配） |
| 生态 Ecosystem | 50% | 5 亿 MC | 含 genesis 切 10%（1 亿 MC）给 depin 模块 |

> **两个结构性盲点（当前 3 池未覆盖）：**
> - ① 无"质押/安全激励池"——固定总量无通胀，staking 奖励只能靠手续费，早期无人愿跑节点。
> - ② 无"直接公开分发/移动节点空投"——85% 锁在团队/社区/生态模块账户，缺群众基础。

## 5. 终版推荐 5 池分配

> 用户锁定了三个数：团队 12%、基金会 13%、早期 5% = **30% 固定**。剩余 **70%** 在设备与质押间分配。终版将 5% 从设备（60→55）挪给质押（10→15），既保留设备最大份额，又补足安全短板。

| 池 | 占比 | 金额（亿 MC） | 释放节奏（关键执行旋钮） |
|---|---|---|---|
| **设备激励 DePIN** | **55%** | 5.5 | 8 年线性，按**真实任务**发放，6 月 cliff；reward 公式需防脚本刷量 |
| **质押/网络安全** | **15%** | 1.5 | 8 年线性滴灌给验证人；**手续费净收入回流此池**补强 |
| 团队 Team | 12% | 1.2 | 1yr cliff + 3yr 线性，3-of-5 多签 |
| 基金会 | 13% | 1.3 | **链上治理支配**；2yr cliff + 5yr 线性 |
| 早期开发/贡献者 | 5% | 0.5 | 1yr cliff + 2yr 线性 |
| **合计** | **100%** | **10.0** | — |

**为什么是 55 / 15 而不是用户原 60 / 10：**
- 设备 55% 仍是**单一最大池**（是质押的 3.6 倍、任何其它池的 4 倍以上），完全满足"设备激励占最大份额"硬要求。
- 质押 10%→**15%（1.5 亿 MC）**，对固定总量链才够撑起早期验证人激励；配合下方回流机制，15% 即可，不必回到激进的 30%。

---

## 6. 让 15% 质押池"够用"的回流机制

光留 15% 不够，须让池子活起来：

1. **手续费回流**：所有 gas 费净值 → 注入质押池，链越用越安全。
2. **设备未领取/罚没回充**：过期未领的设备奖励、作恶罚没 → 回流质押池，防囤积。
3. **治理兜底**：某年验证人收益过低时，治理可从基金会小额划拨补安全。
4. **实现**：质押池需"滴灌分发"模块（周期性把池余额按比例发给验证人），非一次性放出。

---

## 7. 3~5 年长远轨迹

- **第 1~2 年（启动期）**：设备激励把手机节点网络铺开；质押池养活 solo → 少数验证人；基金会靠治理开始拨生态 grant。
- **第 3 年（网络效应期）**：节点规模成形，真实任务费开始产生；设备奖励持续但已线性摊薄；治理完全接管基金会。
- **第 4~5 年（成熟期）**：手续费成为质押安全的重要补充，质押池 + 费双轮驱动；设备网络稳定，"手机全节点"定位坐实。

## 8. 风险旋钮（成败不在数字，在两条执行）

| 旋钮 | 风险 | 对策 |
|---|---|---|
| **设备释放曲线** | 短期放完 → 巨抛压 + 中心化 | 8 年线性 + 按真实任务 + 锁仓，绝不一刀切 lump-sum |
| **质押安全补足** | 固定总量下 15% 仍可能单节点收益被摊薄 | 上述 4 条回流机制 + 治理兜底 |
| **基金会透明度** | 13% 集中金库若不治理，违反可审计原则 | 基金会 13% 动用走链上治理提案，实质变社区控制 |
| **设备防刷量** | 脚本伪造设备任务薅走 5.5 亿 MC | depin reward 公式需真实任务证明 + 速率上限 |
